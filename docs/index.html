// overview

Built for the Jerry.ai Junior Applied Data Scientist application. Jerry's PriceProtect product predicts when customers should re-shop their car insurance — this project demonstrates the same capability end-to-end: data quality analysis, feature engineering, LightGBM classification, and SHAP-powered explainability.

// architecture

raw CSV -> DataQualityReport -> preprocessing pipeline -> LightGBM -> SHAP explainer -> Streamlit app

  • Data Quality First — 8 validation checks with weighted quality score, SQL-style audit queries
  • Preprocessing — ordinal encoding, log transforms, StandardScaler, SMOTE (train-only)
  • Model — LightGBM with GridSearchCV hyperparameter tuning
  • Explainability — SHAP waterfall charts, plain-English factor descriptions, counterfactual tips

// data quality philosophy

This project treats data quality as the primary engineering concern, not model accuracy. The DataQualityReport runs 8 categories of checks — schema validation, missing values, class balance, cardinality, range violations, duplicate detection, suspicious pattern flagging, and a weighted composite score — before any modeling begins.
